# SunPilot 代码文件职责索引

更新日期：2026-06-28

本索引只描述当前 `src` 真实结构。`dist`、`*.tsbuildinfo` 和 `node_modules` 是生成内容，不应作为修改入口。

## 根目录

| 路径                                 | 职责                                 |
| ------------------------------------ | ------------------------------------ |
| `package.json`                       | workspace 脚本、Node/pnpm 版本约束   |
| `pnpm-workspace.yaml`                | `packages/*` 与 `tests/*` 工作区     |
| `docker-compose.yml`                 | 本地 PostgreSQL 16 + pgvector        |
| `tsconfig.base.json`                 | TypeScript 公共配置                  |
| `eslint.config.js`                   | ESLint 公共配置                      |
| `scripts/link-sun-bin.mjs`           | `pnpm install` 后链接本地 `sun` 命令 |
| `tests/integration/agent-ws.test.ts` | daemon/WS 跨包集成测试               |

## packages/protocol

| 文件                | 职责                                            |
| ------------------- | ----------------------------------------------- |
| `agent-commands.ts` | `chat.send`、run、approval JSON-RPC 参数 schema |
| `agent-events.ts`   | `agent.*` 事件唯一词表与协议版本                |
| `agent-errors.ts`   | Agent 错误合同                                  |
| `message-parts.ts`  | 内容块消息协议                                  |
| `rich-cards.ts`     | Rich card 数据合同                              |
| `schemas.ts`        | approval、Skill manifest 等 Zod schema          |
| `types.ts`          | run、step、skill、artifact 等共享 record 类型   |

协议变化必须先改这里，再改 producer、consumer 和测试。

## packages/storage

| 路径                     | 职责                                                    |
| ------------------------ | ------------------------------------------------------- |
| `src/database/`          | database config/factory/context 类型；只允许 PostgreSQL |
| `src/repositories/`      | 面向上层的 repository 接口                              |
| `src/postgres/`          | PostgreSQL repository 实现和 transaction                |
| `src/migrations/`        | 按编号执行的数据库迁移，当前到 `022`                    |
| `src/testing/in-memory/` | 单元测试用内存 DatabaseContext                          |
| `src/paths.ts`           | `~/.sunpilot` 路径、config.json 和目录初始化            |
| `src/artifacts.ts`       | artifact 文件写入                                       |
| `src/redaction.ts`       | 存储前敏感信息脱敏                                      |

新增持久化实体通常要同时修改：record 类型、repository 接口、Postgres 实现、内存实现、migration 与 export。

## packages/launcher

`src/index.ts` 实现 `start/stop/restart/status/doctor/logs/open`。它读取 runtime token 调诊断接口，使用 PID 文件并在缺失时扫描 daemon 进程。

## 维护规则

1. 修改共享 wire contract 时先改 `protocol`。
2. 修改运行链路时同时检查 `composition-root.ts`，避免“有模块但未装配”。
3. 不直接编辑 `dist`；构建会覆盖。
4. repository 变更必须同步 Postgres 与 in-memory 测试实现。
5. 文档只描述当前运行事实；计划和实验能力必须明确标注。
