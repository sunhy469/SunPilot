# SunPilot 代码文件职责索引

更新日期：2026-06-11（Workflow 清理后更新）

说明：
- 覆盖根配置、脚本、`packages/*`、`tests/integration` 下的代码和工程配置文件。
- 不展开 `node_modules/`、`dist/`、图片、LICENSE、README、lockfile 等。

## 根目录

| 文件 | 职责 |
| --- | --- |
| `package.json` | monorepo 根包配置。 |
| `pnpm-workspace.yaml` | pnpm workspace 包范围声明。 |
| `tsconfig.base.json` | 全仓 TypeScript 基础编译配置。 |
| `eslint.config.js` | 全仓 ESLint flat config。 |
| `prettier.config.js` | 全仓 Prettier 格式化配置。 |
| `docker-compose.yml` | 本地 PostgreSQL 开发数据库。 |
| `scripts/link-sun-bin.mjs` | postinstall 链接 `sun`/`sunpilot` 命令。 |

## packages/protocol

| 文件 | 职责 |
| --- | --- |
| `packages/protocol/src/types.ts` | 共享类型：run、step、event、approval、artifact、memory、skill。Workflow 类型已删除。 |
| `packages/protocol/src/schemas.ts` | Zod schema：run/create、approval decision 等。 |
| `packages/protocol/src/agent-commands.ts` | JSON-RPC 命令 schema/type。 |
| `packages/protocol/src/agent-events.ts` | `agent.*` 事件词表和 payload。 |
| `packages/protocol/src/agent-errors.ts` | 错误码、分类、JSON-RPC 映射。 |

## packages/storage

| 文件 | 职责 |
| --- | --- |
| `packages/storage/src/paths.ts` | 本地路径计算、目录初始化、SunPilotConfig（不再含 workflows 配置）。 |
| `packages/storage/src/artifacts.ts` | Artifact 文件存储、checksum/version。 |
| `packages/storage/src/redaction.ts` | 敏感信息脱敏。 |
| `packages/storage/src/database/database.types.ts` | `DatabaseContext` 聚合接口（不再含 WorkflowRepository）。 |
| `packages/storage/src/database/database.config.ts` | 数据库连接配置。 |
| `packages/storage/src/database/database.factory.ts` | 根据配置创建数据库上下文。 |
| `packages/storage/src/migrations/` | SQL schema 演进脚本（001-012）。 |
| `packages/storage/src/postgres/postgres.database.ts` | Postgres `DatabaseContext` 组合实现（不再含 WorkflowRepository）。 |
| `packages/storage/src/postgres/postgres.migrations.ts` | migration runner。 |
| `packages/storage/src/postgres/postgres.*.repository.ts` | 各 repository 的 Postgres 实现。 |
| `packages/storage/src/repositories/index.ts` | repository interface 统一导出（不再含 workflow.repository）。 |
| `packages/storage/src/repositories/*.repository.ts` | 各类 repository 接口。 |
| `packages/storage/src/testing/in-memory-database.context.ts` | 测试用内存 `DatabaseContext`（不再含 workflow map）。 |

## packages/core

| 文件 | 职责 |
| --- | --- |
| `packages/core/src/index.ts` | 统一导出（不再导出 ToolExecutorBridge/WorkflowToolExecutorAdapter）。 |
| `packages/core/src/llm/` | LLM provider、OpenAI-compatible HTTP streaming。 |
| `packages/core/src/agent/agent.service.ts` | AgentService 主入口。 |
| `packages/core/src/agent-kernel/loop-types.ts` | Agent Loop 核心接口（IntentType 中 `workflow_execution` 已替换为 `automation_execution`）。 |
| `packages/core/src/agent-kernel/agent-loop-engine.ts` | Agent Loop 状态机（不再按 `workflow.` 前缀判断 intent）。 |
| `packages/core/src/agent-kernel/context/context-builder.ts` | 统一上下文构建器。 |
| `packages/core/src/agent-kernel/intent/intent-types.ts` | intent 规则（`workflow_execution` 已替换为 `automation_execution`）。 |
| `packages/core/src/agent-kernel/intent/intent-router.ts` | IntentRouter（LLM 分类不再出现 `workflow_execution`）。 |
| `packages/core/src/agent-kernel/planning/rule-based-planner.ts` | RuleBasedPlanner。 |
| `packages/core/src/agent-kernel/tools/tool-types.ts` | SkillSummary（category 不再含 `"workflow"`）、INTENT_SKILL_MAP。 |
| `packages/core/src/agent-kernel/tools/tool-decision-engine.ts` | ToolDecisionEngine（不再有 workflow 特判；支持全限定 id 匹配）。 |
| `packages/core/src/agent-kernel/safety/permission-policy.ts` | PermissionPolicy（categoryFromSkillId 支持全限定 id）。 |
| `packages/core/src/agent-kernel/execution/execution-orchestrator.ts` | ExecutionOrchestrator（直接注入 SkillToolExecutor；categoryFromSkillId 支持全限定 id）。 |
| `packages/core/src/agent-kernel/execution/skill-tool-executor.ts` | SkillToolExecutor — 统一 skill 执行器（resolveCapability 优先解析全限定 id）。 |
| `packages/core/src/agent-kernel/execution/execution-types.ts` | ToolExecutor 接口、retry、concurrency 常量。 |
| `packages/core/src/agent-kernel/reflection/` | BasicReflectionEngine。 |
| `packages/core/src/agent-kernel/response/` | ResponseComposer。 |
| `packages/core/src/agent-kernel/memory/` | DefaultMemoryWriter、脱敏。 |
| `packages/core/src/agent-kernel/persistence/` | repository-based event sink、run state、approval gate。 |
