# SunPilot 当前工程实现总结

更新日期：2026-06-11（Workflow 清理后更新）

本文档总结当前 SunPilot 工程架构、核心模块边界、运行链路和主要实现状态。

## 1. 项目定位

SunPilot 是一个 daemon-first 的本地业务 Agent 运行时：

- Web Chat 交互入口。
- Agent Loop：理解意图、构建上下文、规划、选工具、执行、审批、反思、回复。
- Skill 生态：开发、安装、扫描、权限控制、执行外部能力（含受控 HTTP）。
- PostgreSQL 持久化：run、event、message、approval、artifact、memory、tool/model call、audit 等。
- CLI launcher：通过 `sun` / `sunpilot` 启停 daemon、打开 Web、查看日志和状态。

当前工程已形成端到端闭环：

```text
Browser Web
  <-> daemon WebSocket JSON-RPC / REST
  <-> core AgentService / AgentKernel
  <-> storage PostgreSQL repositories
  <-> SkillToolExecutor -> SkillRunner -> capability.handler()
```

## 2. 总体运行架构

```text
Browser
  |
  | HTTP / WebSocket
  v
SunPilot daemon (Fastify)
  |
  | JSON-RPC command routing
  v
AgentService (core/src/agent)
  |
  | Agent Loop
  v
agent-kernel
  |       |          |
  |       |          +--> LLM provider -> OpenAI-compatible HTTP SSE
  |       |
  |       +--> SkillToolExecutor -> SkillRunner -> installed skills
  |
  v
Storage repositories -> PostgreSQL
```

Skill 是唯一的工具执行抽象。自动化/复合能力通过 Skill capability 表达。

## 3. Monorepo 包边界

| 包 | 职责 |
| --- | --- |
| `@sunpilot/protocol` | 跨包共享协议：类型、Zod schema、Agent 命令、事件、错误码。 |
| `@sunpilot/storage` | 数据访问层：repository 接口、Postgres 实现、migration、测试用内存 DB。 |
| `@sunpilot/core` | 核心业务：AgentService、AgentKernel、LLM provider。不再依赖 workflow。 |
| `@sunpilot/daemon` | 本地服务进程：Fastify REST、WebSocket、静态 Web、composition root。 |
| `@sunpilot/web` | React + Ant Design Chat-first 前端。 |
| `@sunpilot/launcher` | `sun` CLI：启动、停止、状态、doctor、日志、打开 Web。 |
| `@sunpilot/skill-sdk` | Skill 开发 SDK：defineSkill、capability、SkillContext（含 SkillHttpApi）、测试 helper。 |
| `@sunpilot/skill-runner` | Skill 运行时：扫描、校验、执行、权限控制（含受控 ctx.http）。 |
| `tests/integration` | 跨包集成测试。 |

## 4. 核心概念关系

### Agent

Agent 是动态决策者。入口在 `packages/core/src/agent/agent.service.ts`，内部引擎在 `packages/core/src/agent-kernel/`。

### Skill

Skill 是统一的能力抽象。回答"系统能做什么"。开发标准在 `packages/skill-sdk/`，运行执行在 `packages/skill-runner/`。

关系：

```text
Agent -> ToolDecisionEngine -> Skill catalog (全限定 id) -> SkillToolExecutor -> SkillRunner -> capability.handler()
```

Skill 使用全限定 tool id：`<skill-id>:<capability-name>`（如 `cross-border-ecommerce:video.generate.seedance`）。

## 5. protocol 包

主要文件：

- `types.ts`：run、step、event、approval、artifact、memory、skill 等共享类型。Workflow 类型已删除。
- `schemas.ts`：共享请求 schema。
- `agent-commands.ts`：WebSocket JSON-RPC 命令。
- `agent-events.ts`：`agent.*` 事件词表和 payload。
- `agent-errors.ts`：错误码、分类、JSON-RPC 映射。

## 6. storage 包

`storage` 是数据访问层，core/daemon 依赖 repository 抽象而非直接 SQL。

持久化覆盖：conversation、message、run、step、event、approval、artifact、memory、audit、installed skills、idempotency、tool calls、model calls、run status history。

Workflow repository 及相关类型已删除。`DatabaseContext` 不再包含 `workflows` 字段。

## 7. core 包

### agent vs agent-kernel

```text
agent        = 服务入口 / facade / 生命周期操作
agent-kernel = 内部执行引擎 / Agent Loop 机制
```

agent-kernel 子模块：context、intent（`workflow_execution` 已替换为 `automation_execution`）、planning、tools（不再有 workflow 特判）、safety、execution（直接使用 SkillToolExecutor）、response、memory、persistence。

### llm

OpenAI-compatible provider，支持 HTTP SSE streaming。

## 8. daemon 包

daemon 启动时：
1. 创建 DatabaseContext，运行 PostgreSQL migration。
2. 恢复 interrupted/stale 状态。
3. 初始化 SkillRegistry、SkillRunner。
4. 懒加载 AgentService。
5. 注册 REST routes 和 WebSocket server。
6. 托管 Web dist 静态资源。

不再初始化 WorkflowRegistry。

## 9. WebSocket 与 REST

主要 JSON-RPC method：`chat.send`、`chat.stop`、subscribe/unsubscribe、run cancel/resume/retry、approval approve/reject、ping。

REST 入口：health/ready/config、conversations/messages、runs/events、approvals、artifacts、memory、skills、audit、diagnostics/metrics。`/v1/workflows` 已删除。

## 10. skill-sdk 与 skill-runner

### skill-sdk

提供 `defineSkill()`、Zod schema、`SkillContext`（含 `SkillHttpApi`）、`testSkill()` 测试 helper。

### skill-runner

- 扫描 skill 目录，校验 manifest。
- 动态 import 并执行 capability handler。
- 控制并发、超时、AbortSignal。
- 受控 `ctx.http.request()`：URL 解析、host allow-list 校验、audit、header redaction。
- 写 artifact、memory、audit、event。

## 11. web 包

React Chat-first 前端。主链路：WebSocket `chat.send` → 接收 `agent.*` events → streaming message + timeline + approval + artifact。

## 12. launcher 包

```text
sun / sunpilot
```

启停 daemon、状态、doctor、日志、打开 Web。

## 13. 典型调用链

### Chat 到 Agent 回复

```text
Web ChatComposer
  -> WebSocket chat.send
  -> AgentService.handleChatCommand
  -> AgentLoopEngine.run
  -> ContextBuilder -> IntentRouter -> RuleBasedPlanner
  -> ToolDecisionEngine -> PermissionPolicy / Approval
  -> ExecutionOrchestrator -> SkillToolExecutor -> SkillRunner
  -> ResponseComposer -> LLM streamChat
  -> agent.response.delta events -> Web streaming update
```

### Agent 执行 Skill

```text
Intent requires tool
  -> ToolDecisionEngine selects skill capability (全限定 id)
  -> ExecutionOrchestrator records tool call
  -> SkillToolExecutor delegates to SkillRunner
  -> SkillRunner imports skill, checks permissions, runs handler
  -> artifacts/memory/events/audit persisted
  -> Agent observes result and responds
```

## 14. 当前实现特征

- Skill catalog 使用全限定 id（`<skill-id>:<capability-name>`），避免同名冲突。
- `automation_execution` intent 替换原 `workflow_execution`，无特殊 runtime 分支。
- `ctx.http` 受控 HTTP 已实现，所有外部 API 调用必须通过 manifest network allow-list。
- `protocol` 保持稳定协议定义。
- `storage` 的 repository interface 和 Postgres implementation 已分离。
