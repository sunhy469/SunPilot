# SunPilot 命令使用手册

更新日期：2026-06-28

## 1. 初始化

```bash
corepack enable
pnpm install
docker compose up -d postgres
pnpm build

export SUNPILOT_LLM_API_KEY=你的模型密钥
```

版本要求：Node.js `>=22.22.2 <23`，pnpm `>=11.5.1 <12`。

当前 daemon bootstrap 必须能读取 `SUNPILOT_LLM_API_KEY` 或 `DEEPSEEK_API_KEY`；只设置 `SUNPILOT_DP_LLM_API_KEY` 不足以启动。

## 2. sun CLI

```bash
sun start                       # 后台启动，默认 3737
sun start --foreground          # 前台启动
sun start --port 3740           # 指定端口
sun stop
sun restart [--port 3740]
sun status [--port 3740]
sun doctor [--port 3740]
sun logs
sun logs --lines 200
sun open
```

- `status` 调 `/healthz`；
- `doctor` 调 `/v1/diagnostics` 并自动读取 runtime token；
- `stop` 优先读 PID 文件，失败后扫描 daemon 命令行；
- `open` 使用 `SUNPILOT_WEB_URL`，其次 `SUNPILOT_CONSOLE_URL`，最后协议默认 URL。

本地打开 Web：

```bash
export SUNPILOT_WEB_URL=http://127.0.0.1:3737
sun open
```

## 3. 开发命令

```bash
pnpm dev:daemon        # tsx 前台运行 daemon，默认 3737
pnpm dev:web           # Vite，127.0.0.1:3738

pnpm build
pnpm typecheck
pnpm test
pnpm lint
```

单包：

```bash
pnpm --filter @sunpilot/core test
pnpm --filter @sunpilot/daemon test
pnpm --filter @sunpilot/web test
pnpm --filter @sunpilot/storage build
```

## 4. PostgreSQL

```bash
docker compose up -d postgres
docker compose ps
docker compose logs postgres
docker compose stop postgres
docker compose down
```

保留数据不要加 `-v`。删除 volume 会清空全部 SunPilot 数据。

端口冲突：

```bash
SUNPILOT_POSTGRES_PORT=55432 docker compose up -d postgres
export SUNPILOT_DATABASE_URL=postgresql://sunpilot:sunpilot_dev_password@localhost:55432/sunpilot
```

## 5. 带鉴权的 API 调试

`/healthz`、`/readyz` 无需 token；其他 curl 默认需要 bearer token。

```bash
BASE=http://127.0.0.1:3737
TOKEN="$(<"${SUNPILOT_HOME:-$HOME/.sunpilot}/runtime/token")"
AUTH="Authorization: Bearer $TOKEN"

curl "$BASE/healthz"
curl "$BASE/readyz"
curl -H "$AUTH" "$BASE/v1/diagnostics"
curl -H "$AUTH" "$BASE/v1/models"
curl -H "$AUTH" "$BASE/v1/conversations"
curl -H "$AUTH" "$BASE/v1/runs?limit=20"
curl -H "$AUTH" "$BASE/v1/skills"
curl -H "$AUTH" "$BASE/v1/approvals?status=pending"
```

创建会话和简单 HTTP Chat：

```bash
curl -X POST -H "$AUTH" -H 'Content-Type: application/json' \
  "$BASE/v1/conversations" -d '{"title":"API test"}'

curl -X POST -H "$AUTH" -H 'Content-Type: application/json' \
  "$BASE/v1/chat" -d '{"message":"介绍一下 SunPilot"}'
```

`/v1/chat` 是同步精简接口。模型选择、权限模式、附件和完整流式事件应使用 WebSocket `chat.send`。

## 6. WebSocket 调试

浏览器 WebSocket 无法设置 Authorization header，可在本地调试时使用 query token：

```bash
TOKEN="$(<"${SUNPILOT_HOME:-$HOME/.sunpilot}/runtime/token")" node --input-type=module - <<'NODE'
const token = process.env.TOKEN;
const ws = new WebSocket(`ws://127.0.0.1:3737/v1/ws?token=${encodeURIComponent(token)}`);
ws.addEventListener("open", () => ws.send(JSON.stringify({
  jsonrpc: "2.0", id: "ping-1", method: "ping", params: {}
})));
ws.addEventListener("message", event => {
  console.log(event.data);
  ws.close();
});
NODE
```

常用 JSON-RPC method：

- `chat.send`、`chat.stop`；
- `conversation.subscribe`、`conversation.unsubscribe`；
- `run.subscribe`、`run.unsubscribe`、`run.cancel`、`run.resume`、`run.retry`；
- `approval.approve`、`approval.reject`；
- `ping`。

## 7. Run 诊断

```bash
RUN_ID=run_xxx
curl -H "$AUTH" "$BASE/v1/runs/$RUN_ID"
curl -H "$AUTH" "$BASE/v1/runs/$RUN_ID/trace"
curl -H "$AUTH" "$BASE/v1/runs/$RUN_ID/context"
curl -H "$AUTH" "$BASE/v1/runs/$RUN_ID/model-calls"
curl -H "$AUTH" "$BASE/v1/runs/$RUN_ID/tool-calls"
curl -H "$AUTH" "$BASE/v1/runs/$RUN_ID/status-history"
```

## 8. Skill 管理

```bash
curl -X POST -H "$AUTH" "$BASE/v1/skills/reload"
curl -X POST -H "$AUTH" "$BASE/v1/skills/SKILL_ID/enable"
curl -X POST -H "$AUTH" "$BASE/v1/skills/SKILL_ID/disable"
```

Skill 默认放在 `~/.sunpilot/skills/<skill-id>/`。reload 会重新扫描、写 audit 并清理 embedding cache。

## 9. 临时关闭 token 鉴权

仅本机开发/测试使用：

```bash
SUNPILOT_DISABLE_TOKEN_AUTH=1 pnpm dev:daemon
```

即使关闭 token，带 Origin 的浏览器请求仍受 Origin allowlist 限制。不要在公网部署中关闭鉴权。
