# SunPilot 文件路径与作用说明

更新日期：2026-06-28

## 1. 用户运行目录

默认根目录是 `~/.sunpilot`，可通过 `SUNPILOT_HOME` 整体覆盖。

```text
~/.sunpilot/
  config.json
  artifacts/
  skills/
  logs/
    daemon.log
    audit.log
    skill.log
  cache/
  runtime/
    daemon.pid
    token
  analytics/             # 兼容目录，当前无 DuckDB 运行后端
  vectors/lance/         # 兼容目录，当前向量实际在 pgvector
```

| 路径                 | 当前用途                                                        |
| -------------------- | --------------------------------------------------------------- |
| `config.json`        | server/security/skills/storage 的本地配置；部分字段尚未动态生效 |
| `runtime/daemon.pid` | CLI 停止/重启时优先使用的 PID                                   |
| `runtime/token`      | daemon 本地 bearer token，写入权限 `0600`                       |
| `logs/daemon.log`    | 后台 daemon 日志，`sun logs` 读取它                             |
| `logs/skill.log`     | Skill logger 输出                                               |
| `logs/audit.log`     | 预留/兼容日志；主要 audit 数据在 PostgreSQL                     |
| `artifacts/`         | Agent/Skill 产物文件；下载 API 限制在此目录内                   |
| `skills/`            | daemon 当前实际扫描的 Skill 根目录                              |
| `cache/`             | 本地缓存目录                                                    |

`SunPilotPaths.db` 会产生 `~/.sunpilot/sunpilot.db` 字段，但当前数据库 factory 拒绝 SQLite，真实运行数据不写该文件。

## 2. PostgreSQL 数据

默认连接：

```text
postgresql://sunpilot:sunpilot_dev_password@localhost:5432/sunpilot
```

Docker volume 为 `sunpilot_pg_data`，容器名为 `sunpilot-postgres`。conversation、message、run、event、memory、embedding、trace、approval、artifact metadata、Skill catalog 和 Digital World 状态都在 PostgreSQL。

修改 Docker host 端口时必须同时修改连接串：

```bash
SUNPILOT_POSTGRES_PORT=55432 docker compose up -d postgres
export SUNPILOT_DATABASE_URL=postgresql://sunpilot:sunpilot_dev_password@localhost:55432/sunpilot
```

## 3. 命令路径

根包 `postinstall` 执行 `scripts/link-sun-bin.mjs`，将本地 launcher 链接为：

```text
~/.local/bin/sun
~/.local/bin/sunpilot
```

两者都指向 `packages/launcher/dist/index.js`。如果 shell 找不到命令，确认 `~/.local/bin` 在 `PATH`，并先运行 `pnpm build`。

daemon 可执行入口是 `packages/daemon/dist/main.js`，源码是 `packages/daemon/src/main.ts`。

## 4. 源码与生成目录

```text
packages/*/src/        # 唯一源码入口
packages/*/dist/       # pnpm build 生成，不直接编辑
packages/*/*.tsbuildinfo
packages/*/node_modules/
```

Web 静态产物在 `packages/web/dist/`。daemon 优先使用 workspace 里的该目录，再尝试已安装 `@sunpilot/web` 包内的 `dist`。

## 5. 关键源码路径

| 功能               | 路径                                                                               |
| ------------------ | ---------------------------------------------------------------------------------- |
| daemon 装配        | `packages/daemon/src/composition-root.ts`                                          |
| server/auth/worker | `packages/daemon/src/server.ts`                                                    |
| REST               | `packages/api/src/http/`                                                           |
| WS                 | `packages/api/src/ws/`                                                             |
| Agent 入口         | `packages/core/src/agent/agent-service/`                                           |
| Agent Loop         | `packages/core/src/agent-kernel/agent-loop-engine*`                                |
| Context            | `packages/core/src/agent-kernel/context/`                                          |
| Tool               | `packages/core/src/agent-kernel/tools/`                                            |
| Skill              | `packages/skill-runner/src/`                                                       |
| PostgreSQL         | `packages/storage/src/postgres/`                                                   |
| migrations         | `packages/storage/src/migrations/`                                                 |
| Chat UI            | `packages/web/src/pages/ChatPage/`                                                 |
| Digital World      | `packages/platform/src/digital-world/`、`packages/web/src/features/digital-world/` |

## 7. 不应纳入源码修改的内容

- `dist/`、`node_modules/`、`*.tsbuildinfo`；
- `~/.sunpilot/runtime/token` 与任何 API key；
- Docker PostgreSQL volume；
- 本地 `.claude/settings.local.json` 一类个人配置。

涉及数据库 schema 时新增 migration，不回改已经发布的旧 migration；涉及路径语义时同步 `paths.ts`、launcher、daemon 和本文档。
