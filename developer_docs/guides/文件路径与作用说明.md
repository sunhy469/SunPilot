# SunPilot 文件路径与作用说明

更新日期：2026-06-11（Workflow 清理后更新）

本文档说明 SunPilot 运行、安装、构建、部署过程中涉及的主要文件和目录。

## 1. 路径总览

四类路径：
1. 命令入口路径：`sun` 命令
2. 用户运行数据路径：`~/.sunpilot`
3. 项目源码与构建路径：monorepo 源码
4. 部署路径：Nginx 反向代理

## 2. 命令入口路径

### `~/.local/bin/sun`

`sun` 命令的全局入口，指向 `packages/launcher/dist/index.js`，由 `scripts/link-sun-bin.mjs` 创建。

## 3. 用户运行数据目录

### `~/.sunpilot`

默认运行数据根目录，可通过 `SUNPILOT_HOME` 覆盖。

```text
~/.sunpilot/
├── config.json
├── analytics/
├── artifacts/
├── cache/
├── logs/
│   ├── audit.log
│   ├── daemon.log
│   └── skill.log
├── runtime/
│   └── daemon.pid
├── skills/
└── vectors/
    └── lance/
```

### `~/.sunpilot/config.json`

默认内容：

```json
{
  "version": 1,
  "server": { "host": "127.0.0.1", "port": 3737 },
  "security": { "requireLocalToken": false, "allowLan": false },
  "skills": {
    "directories": ["/home/ubuntu/.sunpilot/skills"],
    "autoReload": true
  },
  "storage": { "home": "/home/ubuntu/.sunpilot" }
}
```

注意：`workflows` 配置段已移除。

### `~/.sunpilot/runtime/daemon.pid`

保存 daemon 后台进程 PID，`sun stop` 依赖它。

### `~/.sunpilot/logs`

| 文件 | 作用 |
| --- | --- |
| `daemon.log` | daemon 启动和运行日志 |
| `audit.log` | 审计日志 |
| `skill.log` | skill 执行日志（已脱敏） |

### `~/.sunpilot/artifacts`

保存 skill 产生的产物文件：`~/.sunpilot/artifacts/runs/<run_id>/<artifact_file>`。

### `~/.sunpilot/skills`

用户本地安装的自定义 skills 目录。

## 4. PostgreSQL 数据路径

当前主数据库由 Docker Compose PostgreSQL 提供（volume: `sunpilot_pg_data`）。

清空本地数据库：

```bash
sun stop
docker compose down -v
docker compose up -d postgres
```

旧 SQLite 文件（`~/.sunpilot/sunpilot.db` 等）是历史遗留，确认不需要后可删除。

## 5. 项目源码与构建路径

### 构建产物目录

```text
packages/web/dist
packages/core/dist
packages/daemon/dist
packages/launcher/dist
packages/protocol/dist
packages/storage/dist
packages/skill-runner/dist
packages/skill-sdk/dist
```

`packages/workflow/dist` 不再存在（包已删除）。

## 6. 重要源码路径

| 路径 | 作用 |
| --- | --- |
| `packages/launcher/src/index.ts` | `sun` CLI |
| `packages/daemon/src/server.ts` | Fastify 主服务 |
| `packages/api/src/http/register-routes.ts` | REST API 路由 |
| `packages/core/src/agent/agent.service.ts` | Agent 服务 |
| `packages/core/src/agent-kernel/` | Agent Loop 引擎 |
| `packages/core/src/llm/` | LLM provider |
| `packages/skill-runner/src/runner.ts` | Skill 执行引擎 |
| `packages/skill-sdk/src/index.ts` | Skill 开发 SDK |
| `packages/storage/src/postgres/` | PostgreSQL repositories |
| `packages/storage/src/migrations/` | 数据库迁移 SQL |
| `packages/web/src/pages/ChatPage/` | Web 主页面 |
| `tests/integration/` | 集成测试 |

## 8. 部署相关路径

Nginx 反向代理目标：`http://127.0.0.1:3737`。WebSocket 必须保留 upgrade headers。
