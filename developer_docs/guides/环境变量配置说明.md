# SunPilot 环境变量配置说明

更新日期：2026-06-25

本文档汇总当前 SunPilot 工程涉及的环境变量、默认值、使用位置和配置注意事项。

## 1. 配置方式

当前项目不会自动加载 `.env` 文件。daemon 由 `sun start` 启动时继承当前 shell 环境变量。

```bash
export SUNPILOT_LLM_BASE_URL=https://api.deepseek.com
export SUNPILOT_LLM_MODEL=deepseek-v4-flash
export SUNPILOT_LLM_API_KEY=你的密钥
export SUNPILOT_DATABASE_URL=postgresql://sunpilot:sunpilot_dev_password@localhost:5432/sunpilot
sun stop && sun start
```

## 2. 环境变量总览

### 2.1 核心配置

| 变量名 | 默认值 | 必填 | 作用 |
| --- | --- | --- | --- |
| `SUNPILOT_HOME` | `~/.sunpilot` | 否 | 本地运行数据目录 |
| `SUNPILOT_PORT` | `3737` | 否 | daemon 端口 |
| `SUNPILOT_WEB_URL` | 无 | 否 | `sun open` 打开的地址（优先级高于 `SUNPILOT_CONSOLE_URL`） |
| `SUNPILOT_CONSOLE_URL` | `https://tradeagent.asia` | 否 | Web 控制台默认地址 |
| `SUNPILOT_ALLOWED_ORIGINS` | 空 | 否 | 追加允许的浏览器 Origin（逗号分隔） |
| `SUNPILOT_LOG_LEVEL` | `info` | 否 | Fastify 日志级别（trace/debug/info/warn/error/fatal） |
| `SUNPILOT_DISABLE_TOKEN_AUTH` | 无（默认启用） | 否 | 设为 `1` 可跳过本地 token 鉴权（仅开发环境） |

### 2.2 数据库

| 变量名 | 默认值 | 必填 | 作用 |
| --- | --- | --- | --- |
| `SUNPILOT_DATABASE_PROVIDER` | `postgres` | 否 | 数据库类型（目前仅支持 postgres） |
| `SUNPILOT_DATABASE_URL` | `postgresql://sunpilot:...@localhost:5432/sunpilot` | 否 | PostgreSQL 连接字符串 |
| `SUNPILOT_POSTGRES_PORT` | `5432` | 否 | PostgreSQL 端口（docker-compose 使用） |

### 2.3 LLM 默认配置

以下变量为默认 LLM 配置，当双模型未单独配置时回退到这些值。

| 变量名 | 默认值 | 必填 | 作用 |
| --- | --- | --- | --- |
| `SUNPILOT_LLM_BASE_URL` | `https://api.deepseek.com` | 否 | OpenAI-compatible API base URL（默认） |
| `SUNPILOT_LLM_MODEL` | `deepseek-v4-flash` | 否 | 默认模型名称 |
| `SUNPILOT_LLM_API_KEY` | 无 | 是 | API key（优先级高于 `DEEPSEEK_API_KEY`） |
| `DEEPSEEK_API_KEY` | 无 | 备用 | 备用 API key 变量名 |

### 2.4 双模型配置（DP / Seed）

SunPilot 支持双模型架构：DP 模型负责主要推理，Seed 模型负责辅助推理。
各模型可独立配置 base URL、model、API key。未配置时回退到默认 LLM 配置。

| 变量名 | 默认值 | 必填 | 作用 |
| --- | --- | --- | --- |
| `SUNPILOT_DP_LLM_BASE_URL` | 回退到 `SUNPILOT_LLM_BASE_URL` | 否 | DP 模型 API base URL |
| `SUNPILOT_DP_LLM_MODEL` | 回退到 `SUNPILOT_LLM_MODEL` | 否 | DP 模型名称 |
| `SUNPILOT_DP_LLM_API_KEY` | 回退到 `SUNPILOT_LLM_API_KEY` | 否 | DP 模型 API key |
| `SUNPILOT_SEED_LLM_BASE_URL` | `https://ark.cn-beijing.volces.com/api/v3` | 否 | Seed 模型 API base URL |
| `SUNPILOT_SEED_LLM_MODEL` | `doubao-seed-2-0-lite-260428` | 否 | Seed 模型名称 |
| `SUNPILOT_SEED_LLM_API_KEY` | 无 | 否 | Seed 模型 API key |

### 2.5 Embedding 配置

| 变量名 | 默认值 | 必填 | 作用 |
| --- | --- | --- | --- |
| `SUNPILOT_EMBEDDING_DIMENSIONS` | `1536` | 否 | Embedding 向量维度 |
| `SUNPILOT_EMBEDDING_MODEL` | `text-embedding-3-small` | 否 | Embedding 模型名称 |

### 2.6 Skill 执行配置

| 变量名 | 默认值 | 必填 | 作用 |
| --- | --- | --- | --- |
| `SUNPILOT_SKILL_TIMEOUT_MS` | `300000` (5min) | 否 | 单个 skill 最大执行时间（毫秒） |
| `SUNPILOT_SKILL_MAX_CONCURRENCY` | `4` | 否 | skill 最大并发数 |

### 2.7 速率限制

| 变量名 | 默认值 | 必填 | 作用 |
| --- | --- | --- | --- |
| `SUNPILOT_RATE_LIMIT_WINDOW_MS` | `10000` (10s) | 否 | 速率限制滑动窗口大小（毫秒） |
| `SUNPILOT_RATE_LIMIT_MAX` | `200` | 否 | 每个窗口内最大请求数 |

### 2.8 沙箱与安全

| 变量名 | 默认值 | 必填 | 作用 |
| --- | --- | --- | --- |
| `SUNPILOT_SANDBOX_MODE` | `moderate` | 否 | 工具执行沙箱模式：`strict` / `moderate` / `permissive` |

### 2.9 Memory / RAG 功能开关

以下变量控制 RAG 记忆系统的特性开关。设为 `0` 可禁用对应功能。

| 变量名 | 默认值 | 必填 | 作用 |
| --- | --- | --- | --- |
| `SUNPILOT_MEMORY_MULTIHOP` | 启用 | 否 | 设为 `0` 禁用多跳检索 |
| `SUNPILOT_MEMORY_QUERY_EXPANSION` | 启用 | 否 | 设为 `0` 禁用查询扩展 |
| `SUNPILOT_MEMORY_MMR_RERANK` | 启用 | 否 | 设为 `0` 禁用 MMR 重排序 |

### 2.10 调试

| 变量名 | 默认值 | 必填 | 作用 |
| --- | --- | --- | --- |
| `SUNPILOT_DEBUG_CONTEXT_TIMING` | 无 | 否 | 设为 `1` 启用上下文构建耗时日志 |
| `SUNPILOT_DEBUG_TIMING` | 无 | 否 | 设为 `1` 启用工具决策耗时日志 |

### 2.11 OSS 存储

| 变量名 | 默认值 | 必填 | 作用 |
| --- | --- | --- | --- |
| `SUNPILOT_OSS_SIGNATURE_VERSION` | `v4` | 否 | OSS 签名版本，设为 `v1` 回退到 HMAC-SHA1 签名 |

## 3. 变量详细说明

### `SUNPILOT_HOME`

指定 SunPilot 本地状态目录。影响 config.json、artifacts、skills、logs、cache、runtime 等。

### `SUNPILOT_PORT`

指定 daemon 端口。也可用 `sun start --port 3738` 临时指定。

### `SUNPILOT_CONSOLE_URL`

Web 控制台默认地址。`sun open` 命令优先使用 `SUNPILOT_WEB_URL`，未设置时使用此值。

### `SUNPILOT_ALLOWED_ORIGINS`

追加允许的浏览器 Origin（逗号分隔）。用于 CORS 配置，默认允许 `SUNPILOT_CONSOLE_URL`。

### `SUNPILOT_DISABLE_TOKEN_AUTH`

设为 `1` 跳过本地 daemon token 鉴权。**仅适用于开发环境**，生产环境务必保持默认（启用鉴权）。

### `SUNPILOT_DP_LLM_*` 系列

DP（Dual-Purpose）模型配置，负责主要推理任务。
- `SUNPILOT_DP_LLM_BASE_URL`：API base URL，未设置时回退到 `SUNPILOT_LLM_BASE_URL`
- `SUNPILOT_DP_LLM_MODEL`：模型名称，未设置时回退到 `SUNPILOT_LLM_MODEL`
- `SUNPILOT_DP_LLM_API_KEY`：API key，未设置时回退到 `SUNPILOT_LLM_API_KEY`

### `SUNPILOT_SEED_LLM_*` 系列

Seed 模型配置，负责辅助推理（如 planning、reflection）。默认使用豆包 Seed 2.0 Lite。
- `SUNPILOT_SEED_LLM_BASE_URL`：默认 `https://ark.cn-beijing.volces.com/api/v3`
- `SUNPILOT_SEED_LLM_MODEL`：默认 `doubao-seed-2-0-lite-260428`
- `SUNPILOT_SEED_LLM_API_KEY`：Seed 模型 API key

### `SUNPILOT_EMBEDDING_DIMENSIONS`

Embedding 向量维度，默认 1536（OpenAI text-embedding-3-small 维度）。pgvector 表结构依赖此值。

### `SUNPILOT_EMBEDDING_MODEL`

Embedding 模型名称，默认 `text-embedding-3-small`。仅在已配置 API key 时生效，否则回退到关键词/hash 向量。

### `SUNPILOT_SKILL_TIMEOUT_MS`

单个 skill capability 最大执行时间（毫秒），默认 5 分钟。超时后 runner 会 abort 并写 audit log。

### `SUNPILOT_SKILL_MAX_CONCURRENCY`

skill runner 最大并发执行数，超出上限的执行等待空闲 slot。

### `SUNPILOT_RATE_LIMIT_WINDOW_MS` / `SUNPILOT_RATE_LIMIT_MAX`

HTTP API 速率限制配置。默认每 10 秒窗口最多 200 个请求（按 IP 计数）。

### `SUNPILOT_SANDBOX_MODE`

工具执行沙箱模式：
- `strict`：严格限制文件系统、网络、Shell 命令
- `moderate`（默认）：平衡安全与可用性
- `permissive`：宽松限制，仅阻止最危险的操作

### `SUNPILOT_DATABASE_URL`

PostgreSQL 连接字符串。本地开发默认配合 `docker-compose.yml` 使用。

本地 compose 使用 `pgvector/pgvector:pg16` 镜像。存储迁移 `014_pgvector_embeddings.sql`
会执行 `CREATE EXTENSION IF NOT EXISTS vector` 并创建 `vector(1536)` 字段，因此不能使用
未安装 pgvector 的普通 `postgres` 镜像。

如果本地已经用旧镜像创建过容器，先重建容器再启动：

```bash
docker compose down
docker compose pull postgres
docker compose up -d postgres
```

### `SUNPILOT_LLM_API_KEY`

OpenAI-compatible 模型服务 API key。缺失时尝试读取 `DEEPSEEK_API_KEY`。优先级：`SUNPILOT_LLM_API_KEY` > `DEEPSEEK_API_KEY`。

### Memory / RAG 功能开关

- `SUNPILOT_MEMORY_MULTIHOP=0`：禁用多跳检索（跨记忆关联查找）
- `SUNPILOT_MEMORY_QUERY_EXPANSION=0`：禁用查询扩展（同义词/相关词扩展）
- `SUNPILOT_MEMORY_MMR_RERANK=0`：禁用 MMR 重排序（最大边际相关性）

### `SUNPILOT_OSS_SIGNATURE_VERSION`

OSS 对象存储签名版本。默认 `v4`（HMAC-SHA256）。设为 `v1` 回退到 HMAC-SHA1 签名（仅兼容旧 OSS 服务）。

## 4. Skill 内部可读取的环境变量

Skill 通过 `ctx.secrets.get(name)` 读取环境变量，必须满足：
- manifest 中声明 `permissions.env.allow`
- 变量名在 allow list 中

安全规则：
- 未声明的 env 变量不可读
- 读取 secret 会写入 audit log
- 日志和存储会做敏感信息 redaction

## 5. 推荐服务器配置

```bash
# 默认 LLM（DeepSeek）
export SUNPILOT_LLM_BASE_URL=https://api.deepseek.com
export SUNPILOT_LLM_MODEL=deepseek-v4-flash
export SUNPILOT_LLM_API_KEY=你的密钥

# DP 模型（可回退到默认 LLM，通常无需单独配置）
# export SUNPILOT_DP_LLM_BASE_URL=https://api.deepseek.com
# export SUNPILOT_DP_LLM_MODEL=deepseek-v4-flash
# export SUNPILOT_DP_LLM_API_KEY=你的密钥

# Seed 模型（辅助推理，默认使用豆包）
# export SUNPILOT_SEED_LLM_API_KEY=你的豆包密钥

# Embedding
# export SUNPILOT_EMBEDDING_MODEL=text-embedding-3-small
# export SUNPILOT_EMBEDDING_DIMENSIONS=1536

# 数据库
export SUNPILOT_DATABASE_URL=postgresql://sunpilot:sunpilot_dev_password@localhost:5432/sunpilot

# 沙箱模式（生产环境建议 strict）
export SUNPILOT_SANDBOX_MODE=moderate
```

如有其他域名：

```bash
export SUNPILOT_ALLOWED_ORIGINS=https://your-domain.example
```
