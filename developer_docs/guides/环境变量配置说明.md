# SunPilot 环境变量配置说明

更新日期：2026-06-28

来源：`packages/core/src/config/env.ts`、LLM provider、storage factory、launcher 和 OSS client。

## 1. 配置原则

SunPilot 不自动读取仓库 `.env`。请在启动 daemon 的 shell、systemd/容器环境或部署平台中注入变量。密钥不要写入 Git、`config.json` 或开发者文档。

数字变量由 Zod 在 daemon 启动时校验，非法值会直接启动失败。

## 2. 核心与网络

| 变量                            | 默认值                    | 说明                                          |
| ------------------------------- | ------------------------- | --------------------------------------------- |
| `SUNPILOT_HOME`                 | `~/.sunpilot`             | 本地运行目录                                  |
| `SUNPILOT_PORT`                 | `3737`                    | launcher/daemon 默认端口；`sun --port` 可覆盖 |
| `SUNPILOT_WEB_URL`              | 未设置                    | `sun open` 首选 URL                           |
| `SUNPILOT_CONSOLE_URL`          | `https://tradeagent.asia` | `sun open` 次选 URL                           |
| `SUNPILOT_ALLOWED_ORIGINS`      | 空                        | 逗号分隔的额外浏览器 Origin                   |
| `SUNPILOT_LOG_LEVEL`            | `info`                    | Fastify 日志级别                              |
| `SUNPILOT_DISABLE_TOKEN_AUTH`   | 未设置                    | 设为 `1` 关闭 token，仅限本地开发             |
| `SUNPILOT_RATE_LIMIT_WINDOW_MS` | `10000`                   | 单 IP 滑动窗口                                |
| `SUNPILOT_RATE_LIMIT_MAX`       | `200`                     | 窗口内最大请求数                              |

`SUNPILOT_WEB_URL` 只影响 launcher 打开的地址，不自动改变 daemon bind host 或 CORS。daemon 当前默认只监听 `127.0.0.1`。

## 3. PostgreSQL

| 变量                         | 默认值                                                                | 说明                         |
| ---------------------------- | --------------------------------------------------------------------- | ---------------------------- |
| `SUNPILOT_DATABASE_PROVIDER` | `postgres`                                                            | 只能是 `postgres`            |
| `SUNPILOT_DATABASE_URL`      | `postgresql://sunpilot:sunpilot_dev_password@localhost:5432/sunpilot` | daemon 实际连接串            |
| `SUNPILOT_POSTGRES_PORT`     | `5432`                                                                | Docker Compose host 映射端口 |

`SUNPILOT_POSTGRES_PORT` 不会自动重写连接串。端口变化时必须同时设置 `SUNPILOT_DATABASE_URL`。

## 4. LLM 与双模型

### 4.1 当前启动必需变量

`createDefaultLlmProvider()` 当前只从以下变量取得首个 provider：

| 变量                    | 默认值                     | 说明                                 |
| ----------------------- | -------------------------- | ------------------------------------ |
| `SUNPILOT_LLM_API_KEY`  | 无                         | 推荐设置；当前 daemon bootstrap 必需 |
| `DEEPSEEK_API_KEY`      | 无                         | shared key 的兼容别名                |
| `SUNPILOT_LLM_BASE_URL` | `https://api.deepseek.com` | shared/DP endpoint                   |
| `SUNPILOT_LLM_MODEL`    | `deepseek-v4-flash`        | shared/DP model                      |

至少设置 `SUNPILOT_LLM_API_KEY` 或 `DEEPSEEK_API_KEY`。

### 4.2 DP route override

| 变量                       | 默认值          | 说明         |
| -------------------------- | --------------- | ------------ |
| `SUNPILOT_DP_LLM_API_KEY`  | shared key      | DP route key |
| `SUNPILOT_DP_LLM_BASE_URL` | shared base URL | DP endpoint  |
| `SUNPILOT_DP_LLM_MODEL`    | shared model    | DP model     |

兼容性提醒：只设置 `SUNPILOT_DP_LLM_API_KEY` 仍无法通过前面的 daemon bootstrap。当前部署应同时设置 `SUNPILOT_LLM_API_KEY`；这是代码待统一项。

### 4.3 Seed / Volcengine Ark

| 变量                         | 默认值                                     | 说明                           |
| ---------------------------- | ------------------------------------------ | ------------------------------ |
| `SUNPILOT_SEED_LLM_API_KEY`  | 无                                         | 设置后启用 Seed route          |
| `SUNPILOT_SEED_LLM_BASE_URL` | `https://ark.cn-beijing.volces.com/api/v3` | Ark OpenAI-compatible endpoint |
| `SUNPILOT_SEED_LLM_MODEL`    | `doubao-seed-2-0-lite-260428`              | Seed model                     |

未设置 Seed key 时 `/v1/models` 仍返回 Seed 项，但 `available` 为 `false`。

## 5. Embedding 与意图路由

| 变量                                  | 默认值 | 说明                                              |
| ------------------------------------- | ------ | ------------------------------------------------- |
| `SUNPILOT_EMBEDDING_DIMENSIONS`       | `1536` | embedding 维度，须与 pgvector schema/模型一致     |
| `SUNPILOT_EMBEDDING_MODEL`            | 未设置 | embedding model override                          |
| `SUNPILOT_INTENT_EMBEDDING_THRESHOLD` | `0.95` | embedding 意图短路阈值，运行时 clamp 到 0.75–0.98 |

Embedding provider 复用 shared LLM API key 配置。无法创建真实 provider 时会回退 keyword/hash vector；这能保持功能运行，但不等于真实语义召回质量。

## 6. Skill 与安全

| 变量                             | 默认值     | 说明                             |
| -------------------------------- | ---------- | -------------------------------- |
| `SUNPILOT_SKILL_TIMEOUT_MS`      | `300000`   | 单 capability 超时               |
| `SUNPILOT_SKILL_MAX_CONCURRENCY` | `4`        | 全局 Skill 最大并发              |
| `SUNPILOT_SANDBOX_MODE`          | `moderate` | `strict / moderate / permissive` |

Sandbox mode 是应用策略，不是 OS 隔离。Skill manifest 自己还声明 filesystem/network/env allowlist。

## 7. Aliyun OSS

OSS client 只有在以下五项都存在时才启用：

| 变量                             | 必需 | 说明                                               |
| -------------------------------- | ---- | -------------------------------------------------- |
| `ALIYUN_OSS_ENDPOINT`            | 是   | 例如 `oss-cn-shanghai.aliyuncs.com`，不要带 bucket |
| `ALIYUN_OSS_BUCKET_NAME`         | 是   | bucket 名                                          |
| `ALIYUN_OSS_PUBLIC_BASE_URL`     | 是   | 对象公开访问前缀，建议 HTTPS                       |
| `ALIYUN_OSS_ACCESS_KEY_ID`       | 是   | AccessKey ID                                       |
| `ALIYUN_OSS_ACCESS_KEY_SECRET`   | 是   | AccessKey Secret                                   |
| `SUNPILOT_OSS_SIGNATURE_VERSION` | 否   | 默认 `v4`；仅兼容时设 `v1`                         |

当前 upload prefix 固定为 `sunpilot/uploads`，最大文件 50MB，presigned PUT 有效期 10 分钟；没有对应环境变量覆盖。

示例：

```bash
export ALIYUN_OSS_ENDPOINT=oss-cn-shanghai.aliyuncs.com
export ALIYUN_OSS_BUCKET_NAME=your-bucket
export ALIYUN_OSS_PUBLIC_BASE_URL=https://your-bucket.oss-cn-shanghai.aliyuncs.com
export ALIYUN_OSS_ACCESS_KEY_ID=...
export ALIYUN_OSS_ACCESS_KEY_SECRET=...
```

## 8. 调试

| 变量                    | 默认值 | 说明                                         |
| ----------------------- | ------ | -------------------------------------------- |
| `SUNPILOT_DEBUG_TIMING` | 未设置 | 设为 `1` 输出 native tool loop 细粒度 timing |

## 9. 推荐本地配置

```bash
export SUNPILOT_LLM_API_KEY=...
export SUNPILOT_LLM_BASE_URL=https://api.deepseek.com
export SUNPILOT_LLM_MODEL=deepseek-v4-flash

# 可选
export SUNPILOT_SEED_LLM_API_KEY=...
export SUNPILOT_WEB_URL=http://127.0.0.1:3737

docker compose up -d postgres
sun start --foreground
```

## 10. 已删除或未实现的旧变量

当前代码没有 `SUNPILOT_MEMORY_*`、`SUNPILOT_RAG_*` 等功能开关，也没有通过环境变量配置 DuckDB/LanceDB。不要从旧部署脚本复制这些变量并认为它们会生效。
